# PowerBuilding Workout Tracker

A full-stack strength training tracker built with FastAPI, Jinja2, Tailwind CSS, and Chart.js. Tracks programs, auto-calculates weights from 1RM percentages, and visualises your progress over time.

---

## Tech Stack

- **Backend:** Python + FastAPI
- **Templates:** Jinja2 + Tailwind CSS (CDN) + vanilla JS
- **Charts:** Chart.js (CDN)
- **Database:** SQLite (local dev) → PostgreSQL (production)
- **Auth:** JWT tokens in httpOnly cookies
- **Hosting:** Render.com free tier

---

## Local Development

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/powerbuilding-app.git
cd powerbuilding-app
```

### 2. Create and activate a virtual environment

```bash
python -m venv venv
source venv/Scripts/activate   # Windows (Git Bash)
# or
source venv/bin/activate        # Mac/Linux
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up environment variables

```bash
cp .env.example .env
```

Open `.env` and set at minimum:

```
SECRET_KEY=<generate with: python -c "import secrets; print(secrets.token_hex(32))">
```

Leave `DATABASE_URL` commented out for local SQLite.

### 5. Run database migrations

```bash
alembic upgrade head
```

### 6. Seed the exercise library and example program

```bash
python seed.py
```

### 7. Start the dev server

```bash
uvicorn app.main:app --reload
```

Visit [http://127.0.0.1:8000](http://127.0.0.1:8000) — register an account and you're in.

## Deployment to Render.com

This repo includes a `render.yaml` that configures everything automatically.

### Prerequisites

- A [Render.com](https://render.com) account (free)
- This repo pushed to GitHub

### Steps

1. **Push to GitHub**

   ```bash
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/powerbuilding-app.git
   git push -u origin main
   ```

2. **Go to [render.com](https://render.com) → New → Blueprint**

3. **Connect your GitHub account** and select this repo

4. Render will read `render.yaml` and automatically:
   - Create a free PostgreSQL database (`powerbuilding-db`)
   - Create a web service (`powerbuilding-app`)
   - Set `DATABASE_URL`, `SECRET_KEY`, and `ENVIRONMENT` for you
   - Run migrations and seed data on first deploy

5. **Click Apply** — the first deploy takes ~3–5 minutes

6. Once deployed, open the live URL and register your account

### Environment Variables (set in Render dashboard)

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string — auto-set from the Render DB |
| `SECRET_KEY` | Random secret for signing JWT tokens — auto-generated by Render |
| `ENVIRONMENT` | Set to `production` — auto-set |
| `ANTHROPIC_API_KEY` | Add when ready to enable AI features (optional) |

> **Important:** Never commit your `.env` file — it's in `.gitignore`.

## Project Structure

```
app/
  main.py           # FastAPI entry point, homepage route
  database.py       # SQLAlchemy session, SQLite/Postgres switch
  models.py         # All SQLAlchemy models
  schemas.py        # Pydantic schemas
  auth.py           # JWT auth, password hashing
  utils.py          # Weight calculation (round to nearest 2.5kg)
  routers/
    auth.py         # /auth/register, /auth/login, /auth/logout
    programs.py     # /programs — builder wizard, CRUD
    workouts.py     # /workouts — view, complete
    prs.py          # /prs — 1RM management
    calendar.py     # /calendar — schedule, reschedule, skip
    exercises.py    # /exercises — library, custom exercises
    progress.py     # /progress — sparklines, accessory bests, volume
  services/
    ai_service.py   # Stub for future AI integration
  templates/        # Jinja2 HTML templates
  static/
    app.js          # Minimal vanilla JS
alembic/            # Database migrations
seed.py             # Exercise library + example program
render.yaml         # Render.com deployment config
```

## Adding AI Features (Future)

The codebase is structured for easy AI integration via `app/services/ai_service.py`. When ready:

1. Add your `ANTHROPIC_API_KEY` to `.env` (and to Render env vars)
2. Implement methods in `AIService` using the Anthropic Python SDK
3. Add endpoints to `app/routers/ai.py`

All historical workout data is retained permanently — never deleted — so long-term pattern analysis will work correctly.
